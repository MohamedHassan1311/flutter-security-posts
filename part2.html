لما بتبعت request للـ API، المهاجم يقدر يوقف نفسه في النص ويشوف كل حاجة حتى لو HTTPS:

في حاجه اسمها : Certificate Pinning
بتقول للتطبيق ثق بس في الـ certificate بتاع الـ SSL
x509 -fingerprint -sha256 -noout

# SHA256 Fingerprint=AA:BB:CC:DD:...
‼️
دايماً حط backup certificate، لو نسيته وجددت الـ cert الأساسي التطبيق هيتعطل لكل الـ users!
💠 Android — Network Security Config
<!-- res/xml/network_security_config.xml --> <network-security-config>
  <domain-config>
    <domain includeSubdomains="true">api.yourapp.com</domain>
    <pin-set>
      <pin digest="SHA-256">AAAAAAA...=</pin>
      <pin digest="SHA-256">BBBBBBB...=</pin> <!-- backup -->
    </pin-set>
  </domain-config>

  <!-- امنع HTTP تماماً -->
  <base-config cleartextTrafficPermitted="false"/>
</network-security-config>
💠 iOS — ATS (App Transport Security)
نفس الفكرة على iOS، بتحطه في Info.plist:
<key>NSAppTransportSecurity</key>
<dict>
  <key>NSAllowsArbitraryLoads</key>
  <false/>

  <!-- استثناء لو عندك domain معين محتاج HTTP -->
  <key>NSExceptionDomasecureHTTPLoads</key>
        <true/>
    </dict>
  </dict>
</dict>
⚠️
لو حطيت NSAllowsArbitraryLoads = true عشان تحل مشكلة مؤقتة، Apple ممكن ترفض التطبيق في الـ review.
