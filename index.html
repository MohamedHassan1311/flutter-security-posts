Part 1 — حماية التوكن والـ API Keys في Flutter
🔑
التوكن — ازاي تحفظه صح؟
المشكلة إن معظم الناس بتحفظ التوكن في SharedPreferences وده بيتخزن كـ plain text على الجهاز:
<!-- أي حد يعمل root يشوفه على طول -->
<map>
    <string name="token">eyJhbGciOiJIUzI1NiIs...</string>
</map>
ودا مينفعش لان كدا اي حد معاه التوكن يقدر يعمل access لل API اكنه المستخدم بالظبط ( في طرق تانيه حبايبنا ال backend ممكن يعملوها زي ان يربطو ال token بال id الجهاز بس دا مش موضوعنا دلوقتي ).
طب ازاي نخزنه ؟
نستخدم flutter_secure_storage اللي بيستخدم Android Keystore و iOS Keychain:
// ✅ flutter_secure_storage
const storage = FlutterSecureStorage(
  aOptions: AndroidOptions(
    encryptedSharedPreferences: true,
  ),
);

// حفظ — قراءة — حذف
await storage.write(key: 'token', value: accessToken);
await storage.read(key: 'token');
await storage.delete(key: 'token');
بتعمل حاجة بسيطة، بدل ما تحفظ التوكن plain text، بتطلب من الـ hardware chip في الجهاز إنه يشفره ويحفظه، والـ chip ده مش أي app أو حد يقدر يوصله غير تطبيقك.
‼️
ملاحظه : hardware chip دي عملوها عشان كدا أصلا مهمتها انها تحفظ وتشقر البيانات الحساسه الخاصه بالتطبيق
بس كدا عرفنا ازاي نخزن ال Token 😄

نيجي بقي الـ API Keys — المشكلة الأكبر
التوكن بيتغير، لكن الـ API Key زي Google Maps بيفضل ثابت، ولو اتعمل expose
طبعا كلنا بنحطه كدا
<meta-data
android:name="com.google.android.geo.API_KEY"android:value="AIzaSyA2xA7d84QkR0..."/>
دلوقتي أي حد يعمل decompile للـ APK يطلعه في ثانية:
apktool d app-release.apk
grep -r "AIzaSy" app-release/
الحل: dart-define
بدل ما تحط الـ key جوه الكود أو الـ Manifest، بتمرره وقت البيلد ازاي ؟
بتبيلد كده
flutter build apk \
  --dart-define=MAPS_KEY=AIzaSyA2xA7...
وبتقراه في الكود كده:
const mapsKey = String.fromEnvironment('MAPS_KEY');
وفي الـ Manifest بتستخدم placeholder:
<meta-data
    android:name="com.google.android.geo.API_KEY"
    android:value="${MAPS_KEY}"/>
وفي android/app/build.gradle:
defaultConfig {
    manifestPlacehoring>
وفي ios/Runner/AppDelegate.swift:
let mapsKey = Bundle.main.object(forInfoDictionaryKey: "GoogleMapsApiKey") as? String ?? ""
GMSServices.provideAPIKey(mapsKey)
وفي ios/Runner.xcconfig بتضيف:
MAPS_KEY=$(MAPS_KEY)
والـ build command بيفضل:
flutter build ipa \
  --dart-define=MAPS_KEY=AIzaSyA2xA7...
لو بتستخدم GitHub Actions
# .github/workflows/build.yml
- name: Build APK
  run: |
    flutter build apk \
      --dart-define=MAPS_KEY=${{ secrets.MAPS_KEY }} \
      --dart-define=API_BASE_URL=${{ secrets.API_BASE_URL }}
